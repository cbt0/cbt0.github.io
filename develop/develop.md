# 🛠️ CBT 웹 애플리케이션 종합 개발 및 분석 문서 (develop.md)

이 문서는 **CBT 문제풀이 웹 애플리케이션**의 전체적인 아키텍처, 폴더 구조, HTML/CSS/JS 개별 파일의 핵심 로직 및 코드 분석 결과와 프로젝트의 역사(V0.0 ~ V1.961)를 총망라한 마스터 개발 문서입니다.

## 📁 1. 전체 폴더 및 파일 구조 (Folder Structure)

## ⚙️ 2. 핵심 글로벌 상태 관리 (State Management)

`js/app.js`에서 애플리케이션의 모든 동적 상태는 **`state` 전역 객체**를 통해 실시간으로 단방향/양방향 관리됩니다.

```javascript
const state = {
    exams: {},              // 비동기로 로딩된 과목별 기출문제 데이터 (예: { gas: [...] })
    activeSubject: 'home',  // 현재 활성화된 과목 코드 ('home', 'gas', 'energy_master' 등)
    activeRound: null,      // 현재 활성화되어 풀이 중인 회차 객체 (subject, year, round, questions)
    activeQuestionIndex: 0, // 현재 풀고 있는 문제 인덱스 (0 ~ 59)
    userAnswers: {},        // 사용자가 입력한 답안 저장소 { questionIndex: selectedOptionNumber(1~4) }
    quizMode: 'solving',    // 실행 모드 ('solving': 문제 풀이 중, 'review': 제출 후 오답 확인)
    timerInterval: null,    // 시험 타이머 작동을 위한 setInterval 객체
    timeSpentSeconds: 0,    // 타이머 누적 소요 시간 (초 단위)
    currentQuestions: [],   // 현재 시험지에 출제된 문제 객체 배열
    questionFilter: 'all',  // 문제 목록 필터링 상태 ('all': 전체, 'wrong': 오답만, 'unanswered': 미풀이만)
    currentUser: null,      // 로그인 성공 시 저장되는 사용자 고유 ID
    autoLogoutMinutes: 30   // 유휴 시간 자동 로그아웃 시간 설정 (기본값: 30분)
};
```

## 🔍 3. 파일별 세부 코드 및 기능 로직 분석

### 📄 A. HTML 엔트리 및 SPA 레이아웃 (`index.html`)

1. **SPA 뷰 섹션 제어**
   - `<section>` 태그에 `class="view-section"`을 지정하여 `home-screen`, `rounds-screen`, `quiz-screen`, `grading-screen`, `settings-screen`을 정의합니다.
   - 라우터의 해시 상태에 따라 클래스 `active-view`가 동적으로 제어되어 화면 전환이 이루어집니다.
2. **로그인 / 환영 위젯**
   - `#login-form-container`: 로그인 전 입력 폼. 아이디 저장 체크박스(`save-id-check`)가 내장되어 있습니다.
   - `#welcome-container`: 로그인 성공 후 표시되는 환영 메시지 및 세션 **이어하기 버튼(`home-resume-btn`)** 영역입니다.
3. **퀴즈 레이아웃**
   - `aside.quiz-sidebar`: OMR 답안 마킹 현황판으로, 진행도 텍스트(`quiz-progress-text`)와 마킹 그리드(`#marking-sheet`) 및 최종 제출 버튼이 포함되어 있습니다.
   - `article.quiz-main`: 문제 및 지문 영역으로, 상단 메타 바(필터 드롭다운, 타이머, 계산기 버튼, 제출 버튼), 문제 카드, 1~4번 보기 컨테이너, 이전/힌트/다음 버튼, 그리고 동적으로 열리는 해설 상자(`#explanation-box`)가 배치되어 있습니다.
4. **인터랙티브 모달 윈도우**
   - **결과 리포트 모달 (`#result-modal`)**: 합격 여부 배지, SVG 원형 링 진행바 애니메이션, 세부 통계(소요시간, 맞은 문항, 틀린 문항), 오답 확인 및 완료 버튼을 제공합니다.
   - **계산기 모달 (`#calculator-modal`)**: 드래그 핸들러(`calculator-header`), 공학용 연산자(루트, 자연로그, 상용로그, 지수, 괄호), 그리고 상수 입력 팝업 버튼(`calc-constant-btn`)을 포함합니다.
   - **자주 쓰는 상수 모달 (`#constant-modal`)**: 계산기 내부에서 원주율, 중력가속도, 비열, 잠열, 압력/에너지 환산 상수를 원클릭으로 주입하도록 돕는 불투명 팝업입니다.
   - **문제 점프 모달 (`#question-jump-modal`)**: 문제 번호 뱃지 클릭 시 60문항의 그리드를 띄워 원하는 문항으로 바로 순간이동하는 팝업입니다.

### 📄 B. CSS 디자인 시스템 및 반응형 미디어 쿼리 (`css/style.css`)

1. **디자인 테마 및 CSS Variables**
   - 기본적으로 Slate Dark 계열 바탕색(`--bg-primary: #090d16`, `--bg-secondary: #121824`)에 인디고 바이올렛(`--primary: #6366f1`), 에메랄드 그린(`--success: #10b981`), 라디언트 레드(`--danger: #ff4444`) 포인트를 둔 Premium Dark 테마를 적용했습니다.
   - 테마 변수는 토글 시 `[data-theme="light"]` 변수 그룹으로 스왑되어 라이트 모드로 실시간 변경됩니다.
2. **글라스모피즘(Glassmorphism) 효과**
   - 카드와 모달 배경에 `backdrop-filter: blur(12px)`와 반투명 흰색 테두리(`rgba(255, 255, 255, 0.07)`)를 부여하여 깊이감 있는 세련된 UI를 연출합니다.
3. **모바일 퍼스트 및 반응형 뷰포트 전환**
   - **데스크톱 (992px 초과)**: 퀴즈 화면 진입 시 OMR 마킹판(320px)과 문제지(1fr)가 좌우 2단으로 고정 배치되어 넓은 시야를 제공합니다.
   - **모바일 (992px 이하)**: OMR 마킹판(`quiz-sidebar`)과 문제지(`quiz-main`)가 독립된 탭(`show-quiz-main` / `show-quiz-sidebar`)으로 자동 격리되어 1단으로 스왑 노출됩니다.
   - **폰트 및 여백 최적화 (768px 이하)**: 외곽 여백을 최소화(`10px`)하고, 문제 및 보기 폰트 크기를 `18px`로 확장하고, 보기 내부 패딩을 `10px 10px`로 축소해 스마트폰 가독성을 대폭 끌어올렸습니다.
   - **극소형 스마트폰 (480px 이하)**: 상단 네비게이션 탭의 텍스트(`nav-text`)를 감추고 큼직한 아이콘만 노출하여 뷰포트를 확보합니다.

### 📄 C. 핵심 비즈니스 로직 및 알고리즘 (`js/app.js`)

1. **SPA 해시 라우팅 엔진 (`router()`)**
   - URL 해시(#home, #rounds/..., #quiz, #grading, #settings)의 변화를 감지하여 탭 스타일과 활성화된 뷰를 갱신합니다.
   - **로그인 세션 보호**: 로그인되지 않은 상태에서 특정 해시로 접근 시 자동으로 `#home` 리다이렉트합니다.
   - **타이머 안전장치**: 퀴즈 영역을 벗어나 홈이나 목록으로 복귀할 경우 자동으로 백그라운드 타이머(`timerInterval`)를 무력화하고 상태를 안전하게 격리합니다.
2. **전역 유휴 시간 자동 로그아웃 (`resetIdleTimer()`)**
   - 로그인 직후부터 30분 제한 백그라운드 타이머가 시작됩니다.
   - 마우스 이동, 키보드 입력, 모바일 터치 등 사용자의 행위가 감지될 때마다 타이머가 초기화되며, 30분간 조작이 없을 시 세션을 지키기 위해 강제 로그아웃(`logout()`)을 유도합니다.
3. **실시간 자동 저장 및 이어하기 (`autoSaveSession()`)**
   - 사용자가 시험 시작, 문제 이동, 보기 마킹을 할 때마다 `state` 정보(`activeRound`, `activeQuestionIndex`, `userAnswers`, `timeSpentSeconds`)를 `cbt_${currentUser}_autosave_session` 키로 로컬스토리지에 자동 저장합니다.
   - 로그인 시 자동 세션을 스캔하여 메인 홈에 과목명과 문제 번호가 표시된 이어하기 버튼을 띄우며, 클릭 시 기존 `userAnswers`를 온전하게 보존하여 시험을 재개합니다.
4. **문제 목록 필터링 및 점프 내비게이션 (`doesQuestionMatchFilter()`)**
   - 사용자가 필터(`전체`, `오답`, `미제출`)를 선택하면 하단 OMR 판에서 해당 조건에 맞는 문항 번호만 남기고 숨겨집니다.
   - 이전/다음 문제 버튼 이동 시 `getAdjacentFilteredIndex()` 함수가 인덱스를 순회하며 필터에 맞는 문항이 나올 때까지 숫자를 건너뛰어 자연스러운 탐색 동선을 형성합니다.
5. **게임 스코어 계산 공식 (`submitExam()`)**
   - 사용자가 획득한 기본 점수(100점 만점)에 남은 시간 비례 보너스 점수를 합산하여 최종 게임 점수(`gameScore`)를 산출합니다.
     $$\text{Game Score} = \text{Base Score} + \text{Math.max}\left(0, \left(1 - \frac{\text{소요 시간(분)}}{100}\right) \times 100\right)$$
   - 시간 보너스가 음수가 되어 총점을 갉아먹지 않도록 안전 마진(`Math.max`) 처리가 되어 있습니다.
6. **오답 검습 세션 생성 (`reviewWrongAnswers()`)**
   - '오답 모아 풀기' 클릭 시 로컬스토리지의 해당 과목 오답 이력을 역추적하여 중복을 제거한 뒤, 오직 틀렸던 문항들만으로 재구성한 가상의 `customRound`를 동적으로 생성하여 퀴즈 세션을 새롭게 시작합니다.
7. **게이미피케이션 리더보드 구현 (`renderLeaderboard()`)**
   - 누적 랭킹 목록에서 유저의 최고 득점 기록만을 남기고 중복을 배제합니다.
   - 점수 기준으로 내림차순 정렬하여 1, 2, 3위에게 메달 그라데이션 배지를 부여하고, 본인의 랭킹에는 별도의 '나' 배지를 하이라이트하여 보여줍니다.
8. **계산기 터치/마우스 드래그 앤 드롭 (`dragStart`, `dragMove`)**
   - 모바일 기기의 터치 감지(`touches`)와 PC 브라우저의 마우스 감지 좌표를 통합한 이동 알고리즘을 구현하여, 손가락 터치 시 씹힘 현상이나 웹페이지 화면 스크롤과의 충돌 없이 쾌적하게 계산기를 드래그하여 옮길 수 있습니다.
9. **안전한 수식 연산 및 자동 괄호 완성 (`evaluateCalculatorExpression()`)**
   - 사용자가 입력한 수식의 열린 괄호 수와 닫힌 괄호 수를 대조하여 부족한 만큼 자동으로 수식 끝에 닫는 괄호를 기입해 계산 에러를 방지합니다.
   - `÷`와 `×`를 Javascript 연산자 `/`, `*`로 치환하고, `sqrt`, `ln`, `log` 문구를 `Math` 함수로 컴파일하여 `Function` 생성자를 사용해 독립 연산합니다.
10. **글로벌 에러 로깅 위젯**
    - 스크립트 실행 에러(`window.onerror`) 및 비동기 처리 거부(`unhandledrejection`) 에러 발생 시 에러 정보와 라인 넘버를 로거가 캡처하여, 로그인된 유저의 대시보드 활동 로그 영역에 즉시 에러 기록을 주입 출력하여 원활한 유지보수를 지원합니다.

## 🛠️ 5. 개발 및 디버깅 가이드 (Troubleshooting)

### ① 로컬 환경 파일 실행 시 CORS 에러 주의
웹 사이트가 비동기(`fetch`)로 `data/gas/gas_questions.json`을 읽어오기 때문에, 로컬에서 브라우저로 `index.html` 파일을 더블클릭(`file://` 프로토콜)하여 실행하면 **CORS 정책에 의해 데이터 로딩이 차단**되며 기출문제 목록에 무한 로딩 스피너가 발생합니다.
* **올바른 로컬 실행 방법**: 작업 디렉토리에서 간단한 로컬 웹 서버를 실행하여 접속해야 합니다.
  ```bash
  # Python 3 이용 (localhost:8000 접속)
  python3 -m http.server 8000
  ```

### ② 대대적인 UI 패치 후 링크나 화면 오동작 시
CSS 및 JS 파일이 모바일 탭 라우팅 구조로 대폭 변경되었습니다. 수정 후 모바일 기기나 시뮬레이터에서 화면 전환이 안 되거나 이전 레이아웃이 깨져서 보인다면 **브라우저의 캐시** 때문입니다.
* **PC**: `Ctrl + F5` 또는 `Cmd + Shift + R`로 강력 새로고침을 실행하세요.
* **모바일**: 브라우저 설정에서 **방문 기록 및 캐시 데이터 삭제** 후 접속하세요.
* **정적 리소스 버저닝**: index.html 내 `<link>` 및 `<script>` 호출부에 캐시 방지 쿼리 `?v=1.1.8` 등을 적용했습니다.

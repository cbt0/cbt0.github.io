## 📱 스마트폰용 UI 개편 계획안

### 1. 상단 헤더 & 네비게이션 탭 개편
* **목표**: 개별 과목 탭을 모두 정리하고, 모바일 핵심 네비게이션인 **[홈(아이콘)], [문제], [채점], [설정(아이콘)]**의 4개 탭 구조로 통합합니다.
* **HTML/JS 변경**:
  * `main-nav` 영역에 홈 아이콘(소개말/과목선택), 문제(현재 활성화된 문제풀이 뷰), 채점(마킹 현황 뷰), 설정(테마 토글 또는 설정) 탭을 정의합니다.
  * 과목 선택은 홈 화면(`home-screen`)의 과목 카드를 통해 진입하도록 일원화합니다.
  * `js/app.js`에서 각 탭 클릭 시 화면에 노출할 영역을 제어하는 `switchToTab` 로직을 추가합니다.
* **CSS 변경**: 모바일(768px 이하)에서 헤더가 세로로 길게 늘어지지 않도록, 한 줄로 깔끔하게 정렬된 모바일 최적화 상단 헤더바(또는 하단 탭바) 형태로 전환합니다.

### 2. 문제 풀이 UI 상단 메타 영역 개선 (Top UI)
* **목표**: "과목명 + 년/회 + 타이머 + 제출 버튼"을 우측에 일렬로 배치합니다.
* **HTML/CSS 변경**:
  * `.quiz-header` 내부 레이아웃을 수정하여 오른쪽 끝에 모바일용 슬림 '제출' 버튼(`quiz-top-submit-btn`)을 추가합니다.
  * 공간 확보를 위해 패딩 및 간격을 슬림하게 조정합니다.
* **JS 변경**: 상단 제출 버튼 클릭 시에도 기존의 시험 제출 프로세스(`submitExam()`)가 실행되도록 연결합니다.

### 3. 채점 탭(답안 마킹 현황) 분리 및 상호작용 구현
* **목표**: PC 화면의 좌측 사이드바였던 '답안 마킹 현황'을 모바일에서는 독립된 '채점' 탭의 전용 화면으로 분리합니다.
* **HTML/CSS 변경**:
  * `.quiz-sidebar`를 별도의 탭 뷰 영역으로 분리하여 모바일 화면 전체를 활용해 시원하게 마킹 현황을 보게 합니다.
  * 모바일 환경에서는 10열 또는 6열 배치의 마킹 버튼 크기를 핑거 터치에 적합한 크기(최소 44px)로 확보합니다.
* **JS 변경**: 마킹 버튼 터치 시, `state.activeQuestionIndex`를 업데이트하고, 즉시 **"문제" 탭으로 자동 전환**하며 해당 문제를 렌더링하도록 이벤트를 확장합니다.

### 4. 문제 풀이 UI 세부 요소 정밀화
* **문제 번호 간소화**: 접두사 `"Q. 01"`에서 `"Q. "`를 삭제하고 순수 문제 번호인 **`"01"`**만 깔끔하게 노출합니다.
* **보기 UI 슬림화 (원체크 제거)**: 
  * 보기 아이콘 영역(`.choice-status-icon`)을 제거하여 가로 텍스트 영역을 더 넓게 확보합니다.
  * 선택 여부 및 정/오답 판정은 보기 버튼 전체의 테두리와 배경색 변화(초록색/빨간색)로 충분히 식별 가능하므로 불필요한 시각적 요소를 제거합니다.

### 5. 프레임 마진 최소화 및 폰트 크기 극대화 (CSS)
* **마진/패딩 최소화**: 모바일 해상도(768px 이하)에서 외곽 컨테이너(`.app-content`, `.quiz-container`, `.question-card`)의 패딩을 대폭 축소(기존 24px~40px -> 8px~12px)하여 여백을 없애고 콘텐츠 표시 영역을 넓힙니다.
* **폰트 최적화**: 지문 텍스트의 `font-size`를 모바일 기본 16px에서 18px로, 보기 텍스트를 15px에서 16px로 확대하고, 행간(`line-height: 1.6`)을 확보하여 스마트폰 환경에서의 가독성을 보장합니다.

  - 문제를 풀고 있는 도중('solving' 모드)에는 채점 결과를 미리 확인하지 못하도록 '채점' 탭(답안 마킹 현황)을 숨김 처리.
  - 시험이 끝난 뒤 오답 확인 모드('review' 모드)로 진입할 때만 '채점' 탭이 활성화되어 정오답 표시와 함께 기동되도록 동적 노출 설계.

### 📱 스마트폰용 UI 개편 계획안 및 해결 방식

#### 1. 상단 헤더 & 네비게이션 탭 개편 및 동적 노출 시스템
* **목표**: 개별 과목 탭을 모두 정리하고, 모바일 핵심 네비게이션인 **[홈(아이콘)], [문제], [채점], [설정(아이콘)]**의 4개 탭 구조로 통합하되, 현재 진행 상태에 따라 동적으로 가시성을 관리합니다.
* **HTML/JS 변경**:
  - `main-nav` 영역에 홈 아이콘(소개말/과목선택), 문제(현재 활성화된 문제풀이 뷰), 채점(마킹 현황 뷰), 설정(테마 토글) 탭을 정의했습니다.
  - 과목 선택은 홈 화면(`home-screen`)의 과목 카드를 통해 진입하도록 일원화했습니다.
  - `js/app.js`에서 각 탭 클릭 시 화면에 노출할 영역을 제어하는 `switchTab` 로직을 추가했습니다.
* **CSS 변경**: 모바일(768px 이하)에서 헤더가 세로로 길게 늘어지지 않도록 한 줄로 정렬된 모바일 최적화 상단 헤더바 형태로 전환했습니다.

#### 2. 문제 풀이 UI 상단 메타 영역 개선 (Top UI)
* **목표**: "과목명 + 년/회 + 타이머 + 제출 버튼"을 우측에 일렬로 배치합니다.
* **HTML/CSS 변경**:
  - `.quiz-header` 내부 레이아웃을 수정하여 오른쪽 끝에 모바일용 슬림 '제출' 버튼(`quiz-top-submit-btn`)을 추가했습니다.
  - 공간 확보를 위해 패딩 및 간격을 슬림하게 조정했습니다.
* **JS 변경**: 상단 제출 버튼 클릭 시에도 기존의 시험 제출 프로세스(`submitExam()`)가 실행되도록 연결했습니다.

#### 3. 채점 탭(답안 마킹 현황) 분리 및 상호작용 구현
* **목표**: PC 화면의 좌측 사이드바였던 '답안 마킹 현황'을 모바일에서는 독립된 '채점' 탭의 전용 화면으로 분리하고, PC 데스크톱 화면에서는 문제지가 왼쪽(1fr), 답안 마킹 현황판이 오른쪽(320px)에 배치되도록 화면을 좌우 반전 개편합니다.
* **HTML/CSS 변경**:
  - `.quiz-sidebar`를 별도의 탭 뷰 영역으로 분리하여 모바일에서는 각각 탭으로 1단 노출하고, PC(992px 초과)에서는 `grid-template-columns: 1fr 320px;`를 통해 문제지가 왼쪽, 마킹판이 오른쪽에 배치되도록 하였습니다.
  - 모바일 환경에서는 마킹 버튼 크기를 핑거 터치에 적합한 크기(최소 44px)로 확보했습니다.
* **JS 변경**: 마킹 버튼 터치 시, `state.activeQuestionIndex`를 업데이트하고 즉시 **"문제" 탭으로 자동 전환**하며 해당 문제를 렌더링하도록 이벤트를 확장했습니다.

#### 4. 문제 풀이 UI 세부 요소 정밀화
* **문제 번호 간소화**: 접두사 `"Q. 01"`에서 `"Q. "`를 삭제하고 순수 문제 번호인 **`"01"`**만 깔끔하게 노출합니다.
* **보기 UI 슬림화 (원체크 제거)**:
  * 보기 아이콘 영역(`.choice-status-icon`)을 제거하여 가로 텍스트 영역을 더 넓게 확보했습니다.
  * 선택 여부 및 정/오답 판정은 보기 버튼 전체의 테두리와 배경색 변화(초록색/빨간색)로 처리했습니다.

#### 5. 프레임 마진 최소화 및 폰트 크기 극대화 (CSS)
* **마진/패딩 최소화**: 모바일 해상도(768px 이하)에서 외곽 컨테이너의 패딩을 대폭 축소(기존 24px~40px -> 8px~12px)하여 여백을 없애고 콘텐츠 표시 영역을 넓혔습니다.
* **폰트 최적화**: 지문 텍스트의 `font-size`를 18px로, 보기 텍스트를 16px로 확대하고 행간(`line-height: 1.6`)을 확보하여 가독성을 보장했습니다.

## ⚙️ 3. 핵심 아키텍처 및 글로벌 상태 관리 (State Management)

애플리케이션의 모든 동적 상태는 `js/app.js` 내의 **`state` 전역 객체**를 통해 관리됩니다.

```javascript
const state = {
    exams: {},              // 비동기로 로드된 과목별 기출 데이터 (예: { gas: [...] })
    activeSubject: 'home',  // 현재 선택된 과목 코드 ('home', 'gas', 'energy_craftsman' 등)
    activeRound: null,      // 현재 선택되어 진행 중인 회차 객체 (year, round, questions)
    activeQuestionIndex: 0, // 현재 풀고 있는 문제 인덱스 (0 ~ 59)
    userAnswers: {},        // 사용자가 입력한 답안 저장소 {질문index: 선택보기번호(1~4)}
    quizMode: 'solving',    // 실행 모드 ('solving': 문제 풀이 중, 'review': 제출 후 오답 확인)
    timerInterval: null,    // 타이머 작동을 위한 setInterval 객체
    timeSpentSeconds: 0,    // 타이머 누적 시간 (초 단위)
    currentQuestions: []    // 현재 시험에 출제된 문제 객체 배열
};
```

## 🔍 4. 전체 문제풀이 앱 핵심 기능 및 로직 분석 (Application Logic)

### A. 기출문제 데이터 로딩 (`loadGasQuestions`)
* **동작 방식**: 페이지 로드 시 즉시 실행되며, `fetch('data/gas/gas_questions.json')`를 통해 가스기능사 문제 파일을 불러와 `state.exams.gas`에 캐싱합니다.
* **예외 처리**: 파일 읽기에 실패하거나 CORS 에러 발생 시 콘솔에 에러를 기록하고, UI에는 로딩 스피너를 유지하여 올바르지 않은 상태 진입을 제한합니다.

### B. 화면 라우팅 엔진 (`navigateTo`, `switchTab`, `showView`)
* **메뉴 이동 (`navigateTo`)**: 과목 카드 터치 시 해당 과목의 회차 목록 화면(`rounds-screen`)을 띄우고, 상단 탭의 visual 포커스를 '홈'으로 고정합니다.
* **모바일 탭 전환 (`switchTab`)**: 스마트폰 화면에서 **[홈]**, **[문제]**, **[채점]**, **[설정]** 탭 전환을 담당합니다. 
* **보호 장치**: 활성화된 시험 정보(`state.activeRound`)가 없을 때 사용자가 상단의 **문제** 또는 **채점** 탭을 클릭하는 경우, alert 창을 띄운 뒤 `switchTab('home')`을 통해 초기 홈 화면으로 강제 전환합니다.
* **뷰 제어 (`showView`)**: 인자로 받은 `viewName`에 해당하는 DOM 요소만 `.active-view` 클래스를 추가하여 `display: block` 처리하고, 나머지 요소는 숨깁니다. 전환 즉시 모바일 스크롤 위치를 맨 위로 올립니다.

### C. 퀴즈 세션 초기화 (`startQuiz`)
사용자가 회차 카드를 클릭하면 퀴즈 세션이 시작됩니다.
1. `state` 초기화: `activeRound` 지정, 답안 기록(`userAnswers`) 비우기, 풀이 모드(`solving`) 설정, 타이머 초 초기화.
2. 화면 텍스트 바인딩: 상단 헤더에 선택한 과목명 및 회차 정보 출력.
3. DOM 초기화: 이전 힌트 박스 닫기, 마킹판 렌더링, 1번 문제 렌더링.
4. 타이머 구동: 기존 타이머 인터벌을 클리어하고 `setInterval`을 이용해 1초마다 `state.timeSpentSeconds`를 증가시키며 화면에 `MM:SS` 형태로 갱신.
5. 탭 강제 전환: `switchTab('quiz')`을 호출하여 문제 풀이 본문으로 전환.

### D. 문제 렌더링 엔진 (`renderActiveQuestion`)
`state.activeQuestionIndex`가 가리키는 문제를 찾아 화면에 동적으로 렌더링합니다.
* **문제 및 지문**: 문제 번호를 2자리 수(`01`)로 가공하여 타이틀과 결합해 표시합니다.
* **보기 바인딩**: 각 보기 버튼(`.choice-item`) 내부의 `.choice-text` 노드를 찾아 기출 데이터 매핑.
* **동적 스타일 복원**: 사용자가 이미 문제를 푼 경우, 선택한 답변과 실제 정답을 분석하여 아래와 같이 장식합니다:
  * 내가 선택한 답이 **정답**인 경우: 해당 보기 버튼에 `.correct` 추가.
  * 내가 선택한 답이 **오답**인 경우: 내가 선택한 버튼에는 `.wrong`, 실제 정답 버튼에는 `.correct`를 추가하여 시각적 오답 노트 제공.
* **힌트 패널**: 사용자가 답을 푼 경우 자동으로 해설 박스(`.explanation-panel`)의 `.collapsed` 클래스를 지워 펼치고, 해설이 없거나 풀지 않은 경우 접힌 상태로 둡니다.
* **이동 제어**: 1번 문제에서는 '이전 문제' 버튼 비활성화, 60번 문제에서는 '다음 문제' 버튼을 비활성화 처리합니다.

### E. 답안 선택 및 반응형 마킹판 (`handleSelectAnswer`, `renderMarkingSheet`, `updateMarkingStatus`)
* **보기 클릭 시 (`handleSelectAnswer`)**:
  * 채점(리뷰) 모드거나 이미 해당 문제를 마킹한 경우에는 다중 클릭을 방지하기 위해 입력을 차단합니다.
  * 사용자가 보기를 선택하면 `state.userAnswers`에 답안을 기록하고 즉시 `renderActiveQuestion()`을 호출해 즉각적인 정오답 피드백(초록/빨강 색상 전환)과 해설 노출을 수행합니다.
  * **전체 완료 알림**: 마킹 개수가 문제 전체 개수(60개)와 같아지면 1초 뒤 피드백 확인 시간을 준 후, 제출 확인창(`confirm`)을 띄워 빠른 제출을 돕습니다.
* **마킹판 드로잉 (`renderMarkingSheet`)**:
  * `state.currentQuestions` 배열 크기만큼 마킹용 원형 버튼(`button.marking-btn`)을 생성합니다. 각 버튼은 ID(`marking-num-idx`)를 부여받아 동적으로 상태를 갱신할 수 있도록 설계되었습니다.
  * 마킹 버튼 클릭 시 해당 문제 인덱스로 이동하며, 모바일 화면에서는 자동으로 **[문제]** 탭으로 전환해 줍니다.
* **마킹판 상태 갱신 (`updateMarkingStatus`)**:
  * 풀이 모드(`solving`) 시: 마킹한 이력이 있는 번호는 `.solved` 스타일을 입혀 마킹 유무를 시각화합니다.
  * 리뷰 모드(`review`) 시: 각 번호에 대해 정답 여부를 판단하여 번호 동그라미 전체를 초록색(`.correct`) 또는 빨간색(`.wrong`)으로 채워 실시간 대시보드를 구축합니다.
  * 상단 진행 표시줄에 `푼 문제 수 / 전체 문제 수` 텍스트를 지속적으로 갱신합니다.

### F. 채점 엔진 및 결과 분석 (`submitExam`)
사용자가 제출하기 버튼을 누르면 작동하는 핵심 평가 로직입니다.
1. `clearInterval(state.timerInterval)`로 타이머를 멈춥니다.
2. `state.currentQuestions`를 순회하며 `state.userAnswers[idx]`값과 `q.answer`값을 1:1 비교하여 정답 갯수를 누적 산출합니다.
3. 100점 만점으로 스케일링한 점수(`scoreVal`)를 내고, 자격증 취득 기준인 **60점 이상**인 경우 합격(`PASS`), 미만인 경우 불합격(`FAIL`) 상태로 나눕니다.
4. 결과 리포트 모달창의 내용 바인딩: 점수, 정답 갯수, 소요시간을 파싱하여 동적으로 업데이트합니다.
5. **원형 그래프 애니메이션**: SVG stroke-dashoffset 기술을 사용하여 점수 비율에 맞춰 게이지가 채워지는 미려한 모션 그래픽을 적용합니다.
   $$\text{dashOffset} = \text{둘레(440)} \times \left(1 - \frac{\text{점수}}{100}\right)$$
6. **진도 및 통계 기록**: `localStorage`에 이번 회차의 풀이 완료 기록(`completed: true`, 점수, 소요 시간)을 개별 키(`cbt_progress_과목_연도_회차`)로 영속화하고, 전체 누적 통계(`cbt_global_stats`)에 반영합니다.
7. 결과 리포트 모달창 활성화(`.active`).

### G. 오답 검토 모드 (`enterReviewMode`)
제출 후 모달창에서 "오답 확인하기"를 누르면 진입하는 모드입니다.
* `state.quizMode = 'review'`로 모드를 전환하고, 처음부터 오답 분석을 할 수 있도록 `activeQuestionIndex = 0`으로 포커싱을 리셋합니다.
* 마킹판을 다시 그려 각 번호에 빨간색/초록색 전구(정오답 표시)를 켜주고, 문제 화면에도 정답 가이드 데코레이션을 동적으로 입힙니다.

### H. 테마 제어 시스템 (`initTheme`, `toggleTheme`)
* 라이트 모드 및 다크 모드를 지원하며, 사용자의 선택 테마를 `localStorage`의 `cbt_theme` 키에 저장합니다.
* `document.documentElement.setAttribute('data-theme', theme)` 구조를 활용하여 CSS 변수(다크 테마 셋/라이트 테마 셋)를 통째로 교체합니다.

## 🛠️ 6. 개발 및 디버깅 가이드 (Troubleshooting)

### ① 로컬 환경 파일 실행 시 CORS 에러 주의
웹 사이트가 비동기(`fetch`)로 `data/gas/gas_questions.json`을 읽어오기 때문에, 로컬에서 브라우저로 `index.html` 파일을 더블클릭(`file://` 프로토콜)하여 실행하면 **CORS 정책에 의해 데이터 로딩이 차단**되며 기출문제 목록에 무한 로딩 스피너가 발생합니다.
* **올바른 로컬 실행 방법**: 작업 디렉토리에서 간단한 로컬 웹 서버를 실행하여 접속해야 합니다.
  ```bash
  # Python 3 이용 (localhost:8000 접속)
  python3 -m http.server 8000
  ```

### ② 대대적인 UI 패치 후 링크나 화면 오동작 시
CSS 및 JS 파일이 모바일 탭 라우팅 구조로 대폭 변경되었습니다. 수정 후 모바일 기기나 시뮬레이터에서 화면 전환이 안 되거나 이전 레이아웃이 깨져서 보인다면 **브라우저의 캐시** 때문입니다.
* **PC**: `Ctrl + F5` 또는 `Cmd + Shift + R`로 강력 새로고침을 실행하세요.
  * **모바일**: 브라우저 설정에서 **방문 기록 및 캐시 데이터 삭제** 후 접속하세요.

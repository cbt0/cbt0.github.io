# Antigravity AI Coding Assistant Rules

## General Behavior Guidelines
- **사용자 확인 우선**: 사용자가 명시적으로 지시하거나 개선을 요구한 사항이 아닌 영역(예: 임의의 UI/UX 스타일 수정, 추가 기능 구현 등)은 개발자가 판단하여 임의로 변경하지 않습니다. 변경이 필요하다고 판단되는 경우, 작업을 수행하기 전에 반드시 사용자에게 질문하여 승인을 먼저 받으십시오.
- **디버깅 시 최근 Git 이력 추적**: 버그가 발생하고 원인이 즉시 파악되지 않는다면, 전체 프로젝트를 무작정 뒤지기 전에 최근 Git 커밋 5개 내외의 변경 사항(diff)을 우선 분석하십시오. 대부분의 버그는 최근 수정 사항에서 비롯됩니다.
- **코드 수정 전 사전 승인 필수**: 개발 진행 시 코드를 실제로 수정하는 도구를 호출하기 전에, 변경하고자 하는 대상 코드와 수정안을 사용자에게 텍스트로 미리 제시하여 동의를 얻은 후 반영하십시오.
